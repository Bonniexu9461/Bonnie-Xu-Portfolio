One of the most frustrating things about working with AI is getting results that are almost right, but not quite. Most of the time, the problem isn't the AI. It's the instruction.

When we say "make the text bigger" or "add some space", we're asking AI to guess what we mean. When we say font-size: 48px or padding-bottom: 80px, there's no guessing. Use this as your cheat sheet.

The formula that works: Attach a screenshot →︎ point to the exact element →︎ use the terms from this guide →︎ tell AI what NOT to change. One clear instruction beats three vague ones.

Typography
Instead of saying…Use this termPrompt you can use
"The heading is too big" →︎ font-size Reduce the H1 font-size to 48px — it's too large on desktop
Always give a specific px value — don't make AI guess
"The font looks too thin / too heavy" →︎ font-weight The heading font-weight is too light — change it to 600 (semibold)
300=light · 400=regular · 600=semibold · 700=bold
"The lines are too close together" →︎ line-height The body text feels cramped — increase the line-height to 1.6 so it's easier to read
Body text: 1.5–1.7 · Headings: 1.1–1.3
"The letters feel too spaced out / cramped" →︎ letter-spacing The heading letters feel too spread out — tighten the letter-spacing to -0.02em
Negative values = tighter. Use on large headings only
"This text is hard to read" →︎ font-size + color The body text is hard to read — increase the font-size to 16px and darken the colour to #333333
Fix both size and contrast at the same time
"Align this text" →︎ text-align Left-align the body text — it's currently centred and hard to scan
"Make the font scale with the screen" →︎ clamp() Use clamp() on the H1 so it scales with the screen — minimum 28px on mobile, maximum 60px on desktop
clamp(min, fluid, max) — no breakpoints needed

Spacing & Layout
Instead of saying…Use this termPrompt you can use
"These two elements are too close" →︎ margin / gap The cards are too close together — increase the gap to 24px between them
gap = space between children · margin = space outside element
"The content is too close to the edge" →︎ padding The text inside the card is too close to the edge — add 24px padding on all sides
padding = space inside the element
"This section feels cramped top and bottom" →︎ padding-top / padding-bottom This section feels too tight — add 80px padding-top and padding-bottom to give it more breathing room
"The content area is too wide" →︎ max-width The content stretches too wide on large screens — set max-width to 1200px and centre it
"Put these cards in a row" →︎ flexbox Lay the cards out in a flex row with a 24px gap — make them wrap to a new line on mobile
"Image on the left, text on the right" →︎ 2-column grid Use a 2-column grid — image takes up the left half, text on the right, 48px gap between them
"The image should stay square / a fixed ratio" →︎ aspect-ratio Lock the image to a square using aspect-ratio: 1/1 so it doesn't distort when the content changes
16/9 for landscape · 4/3 for cards · 1/1 for square
↕
Responsive Design
Mobile-first best practice: Write your base styles for mobile, then use min-width media queries to add complexity as the screen grows. This is cleaner and performs better than overriding desktop styles for mobile.
Instead of saying…Use this termPrompt you can use
"Make it work on mobile" →︎ media query Add a media query for screens below 768px — at that breakpoint, reduce the font size and stack the columns vertically
768px = tablet · 480px = mobile
"Stack it vertically on mobile" →︎ flex-direction: column On mobile (below 768px), switch the row layout to a column so each card stacks vertically
"The font size should adapt to screen size" →︎ clamp() / vw units Make the heading font-size fluid using clamp — 28px minimum on small screens, 60px maximum on desktop, scaling smoothly between
"Cards should auto-fit the row" →︎ auto-fill / auto-fit Set the card grid to auto-fit so cards reflow automatically — each card should be at least 280px wide and fill the remaining row space
"Images should fill their container" →︎ width: 100% + object-fit Make the image fill its container without stretching — set width to 100% and use object-fit: cover
Pair with a fixed height or aspect-ratio to avoid distortion
"The padding should be smaller on mobile" →︎ responsive padding The section padding is too large on mobile — use clamp() so it's 20px on small screens and scales up to 80px on desktop

Visual Style
Instead of saying…Use this termPrompt you can use
"The corners are too sharp" →︎ border-radius The card corners are too sharp — round them with a border-radius of 12px
0=sharp · 8–16=soft cards · 9999px=pill button
"Add a border" →︎ border Add a subtle border to the card — 1px solid, light grey (#E0E0E0)
"Add a shadow" →︎ box-shadow Add a soft box-shadow to the card so it lifts off the background slightly — keep it subtle
Subtle: small blur, low opacity · Heavy: large spread, higher opacity
"This colour is too dark / too light" →︎ opacity / hex colour The secondary text is too dark — lighten it to #666666 or reduce its opacity to 60%
"Add a gradient background" →︎ linear-gradient Replace the flat background with a soft diagonal linear-gradient — light lavender to light blue, top-left to bottom-right
"The image is cropped weirdly" →︎ object-fit / object-position The image is cropping wrong — set object-fit to cover and object-position to center top so the face stays visible

States & Interactions
Instead of saying…Use this termPrompt you can use
"When I hover over the button" →︎ :hover On :hover, reduce the button opacity to 0.8 — add a 0.2s transition so it fades smoothly
"When it's clicked / selected" →︎ :active / .active class When the button is pressed (:active), scale it down slightly to give a physical click feel
"When tabbing through with keyboard" →︎ :focus-visible Add a visible focus ring on :focus-visible for keyboard users — a 2px blue outline with a small offset
Never remove focus styles without a visible replacement
"Make the button look inactive / unavailable" →︎ :disabled Style the :disabled button so it's clearly unavailable — reduce opacity to 40% and change the cursor to not-allowed
"Show a loading spinner state" →︎ loading state When the form is submitting, add a loading state to the button — show a spinner, replace the label text, and disable clicks
"Highlight the current page in the nav" →︎ active state / aria-current Add an active state to the nav link for the current page — darker colour and a 2px underline
→︎
Motion & Transitions
Responsive motion best practice: Always wrap animations in @media (prefers-reduced-motion: no-preference) so users who have motion sensitivity aren't affected.
Instead of saying…Use this termPrompt you can use
"Add a smooth transition when hovering" →︎ transition Add a transition to the button so the opacity change on hover feels smooth — 0.2s is enough
Always specify: property, duration, and easing
"Make it fade in when the page loads" →︎ @keyframes / animation Animate the hero section to fade in when the page loads — use a CSS animation, 0.5s duration, ease timing
"Move it / slide it / scale it" →︎ transform On hover, use transform to lift the card up 4px — don't use top or margin, it'll cause jank
transform = GPU-accelerated, no layout reflow
"The animation feels too fast / too slow" →︎ duration The hover animation feels too quick — slow the transition duration to 0.3s
Micro-interactions: 100–200ms · Page transitions: 300–500ms
"The animation feels mechanical / robotic" →︎ easing / cubic-bezier The card hover feels stiff — change the easing to ease-out so the motion feels more natural
ease-out = decelerates into resting · linear = robotic

Position & Layering
Instead of saying…Use this termPrompt you can use
"The nav should stay at the top when scrolling" →︎ position: fixed Make the navbar fixed at the top so it stays visible while scrolling — make sure it has a high z-index so nothing overlaps it
"The sidebar should stick while I scroll" →︎ position: sticky Make the sidebar sticky so it follows the user as they scroll — it should stop 96px from the top to clear the fixed nav
"Place this badge in the top-right corner of the card" →︎ position: absolute Position the "New" badge in the top-right corner of the card absolutely — 12px from the top, 12px from the right. Make sure the card has position: relative
"The modal / overlay should sit on top of everything" →︎ z-index The modal is appearing behind the nav — give it a higher z-index so it sits on top of everything on the page
Nav: 100 · Dropdown: 200 · Modal: 900 · Toast: 1000
"The content overflows / gets cut off" →︎ overflow The long text is spilling outside the card — add overflow: hidden to clip it, or overflow: auto to add a scrollbar only when needed
